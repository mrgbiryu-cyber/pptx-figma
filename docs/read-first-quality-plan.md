# Read-First 품질개선 계획

## 1. 목적

이 프로젝트의 현재 1차 목적은 `PPT -> Figma` 고급 변환 품질을 끌어올리는 것이다.

품질 기준은 `edit-first`가 아니라 `read-first`로 둔다.

즉, 변환 결과는 먼저 아래를 만족해야 한다.

1. PPT 사용자가 Figma 결과를 보고 바로 이해할 수 있어야 한다.
2. 흐름, 관계, 단계, 강조가 오해 없이 보여야 한다.
3. 최소한의 수정은 가능해야 하지만, 수정 가능성보다 해석 가능성이 우선이다.

## 2. 프로젝트 목적 재확정

이 프로젝트의 목표는 아래 순서로 본다.

1. PPT를 Figma로 고품질 전환한다.
2. 전환 결과를 `node` 구조로 분해한다.
3. 이 결과를 DB에 적재한다.
4. 정책 찾기, 검색, 질의응답이 가능하게 만든다.

현재 단계의 최우선 목표는 `1. PPT -> Figma 고품질 전환`이다.

## 3. 제외 범위

현재 단계에서는 아래 항목을 우선순위에서 제외한다.

- 에셋화
- 컴포넌트 치환
- 디자인 시스템 연계

이유:

- 지금의 핵심 목표가 아니다.
- 구현 복잡도 대비 효과가 불확실하다.
- PPT 사용자가 Figma에서 자연스럽게 이해하고 작업할 수 있는지가 더 중요하다.

## 4. 품질 원칙

### 4-1. Read-First

- 수정 가능성보다 먼저 `보이는가`, `이해되는가`를 본다.
- 흐름이 틀리면 실패다.
- 화살표, 표, 텍스트, 박스 관계가 오해 없이 보여야 한다.

### 4-2. Visual Fidelity 우선

- 원본과 100% 동일할 필요는 없지만, 의미 전달이 달라지면 안 된다.
- 특히 다음은 해석을 방해하면 실패로 본다.
  - 잘못된 화살표 방향
  - 텍스트 줄바꿈 오류
  - 박스 묶음 붕괴
  - 표 가독성 붕괴

### 4-3. 최소한의 수정 가능성 유지

- 텍스트
- 표 셀
- 핵심 박스

위 항목은 가능한 한 수정 가능해야 한다.

다만 화살표처럼 수정이 어렵더라도, 시각적으로 정확하게 읽히는 쪽이 우선이다.

### 4-4. 하드코딩 금지, 패턴 기반 개선

- 개별 객체 보정은 디버깅까지만 허용한다.
- 최종 구현은 유형별 규칙으로 일반화해야 한다.

## 5. 품질개선 범위

### Track A. 텍스트 가독성

목표:

- 줄바꿈
- 크기
- 정렬
- 박스 내부 배치

가 PPT와 해석상 유사해야 한다.

수정 항목:

- 문단/강제 개행 유지
- wrap 판정 강화
- text box inset 보정
- font size 추정 보정
- overflow 방지

통과 기준:

- 사용자가 텍스트를 읽을 때 줄바꿈 때문에 의미를 오해하지 않을 것

### Track B. 화살표 / 흐름 가독성

목표:

- 화살표는 editable보다 먼저 `흐름이 정확히 읽혀야` 한다.

수정 항목:

- 시작점 / 끝점 anchor 규칙
- 직선 / 꺾임 path 규칙
- 세로 직선 누락 방지
- 머리 방향
- 머리와 선 끝 정렬
- 객체의 어느 면에 붙는지 규칙화

통과 기준:

- 사용자가 흐름 방향을 오해하지 않을 것
- 어떤 박스에서 어떤 박스로 가는지 한 번에 읽힐 것

### Track C. 박스 / 도형 정합성

목표:

- 박스, diamond, 그룹이 화면 해석에 맞게 보여야 한다.

수정 항목:

- diamond orientation
- labeled shape 위치
- 그룹 내부 상대 위치
- 불필요한 외곽선 제거
- 박스 묶음 정합성

통과 기준:

- 프로세스 단계와 묶음이 직관적으로 보일 것

### Track D. 표 가독성

목표:

- 표는 semantic 구조뿐 아니라 사람이 읽을 수 있어야 한다.

수정 항목:

- cell width / row height
- merged cell 위치
- 중복 셀 제거
- 텍스트 줄바꿈
- 표 경계선 정합성

통과 기준:

- 셀 의미가 보이고, 표 내용을 읽는 데 불편이 없을 것

## 6. 우선순위

### 1순위

- 1page / 2page 화살표
- 1page / 2page 텍스트 줄바꿈
- 2page 표 가독성

### 2순위

- 1page diamond / 그룹 정합성
- 3page 텍스트/표 기본 안정화

### 3순위

- 3page 종합 품질 개선
- 이후 전체 benchmark 확대

## 7. 마감 기준

### Gate 1-A

대상:

- 1page
- 2page

판정 질문:

- 이 두 페이지는 PPT 사용자가 Figma에서 바로 읽고 이해할 수 있는가?

통과 조건:

- 화살표 방향/연결이 읽힌다
- 텍스트 줄바꿈이 의미를 해치지 않는다
- 표/박스가 크게 깨지지 않는다

### Gate 1-B

대상:

- 3page

판정 질문:

- 복합 UI도 read-first 기준으로 이해 가능한가?

통과 조건:

- 텍스트, 이미지, 표, 그룹이 동시에 무너지지 않는다

## 8. 구현 원칙

### 유지할 것

- intermediate model
- node 구조화
- slide별 비교 검증

### 바꿔야 할 것

- 품질 기준을 `edit-first`에서 `read-first`로 전환
- 화살표를 native 편집성보다 가독성 중심으로 구현
- 시각 품질 판단을 더 강하게 앞세움

### 하지 않을 것

- 에셋화
- 컴포넌트 치환
- 디자인 시스템 연계

## 9. 현재 상태에 대한 판단

- 1page / 2page는 화살표와 일부 텍스트/표 정합성만 보완되면 `PASS` 가능성이 있다.
- 3page는 아직 별도 개선 트랙으로 봐야 한다.
- 공식적으로는 아직 `Figma 운영 가능성 검증 단계`다.

## 10. 최종 확정 문장

이 프로젝트의 현재 품질개선 계획은 아래로 확정한다.

> PPT를 Figma로 고급 전환할 때, 수정 가능성보다 먼저 보이는 결과가 원본과 유사하게 해석되어야 하며, 특히 텍스트, 화살표, 박스, 표의 관계가 명확하게 읽히는 수준까지 품질을 끌어올린 후 다음 단계로 진행한다.
